    \hypertarget{installing-libraries-and-managing-enviroments}{%
\paragraph{Installing libraries, and managing
enviroments}\label{installing-libraries-and-managing-enviroments}}

    Options 1. Install packages with the package manager globally as a super
user or append --user to install in the home directory - Using
\textbf{pip} - sudo pip install pandas - sudo pip install --upgrade
pandas - sudo pip uninstall pandas - It is not recommended if you are
working in multiple projects in the same machine 2. Install the
interpreter in your operating system - sudo pip install virtualenv -
Isolate your working enviroments using virtualenv by creating
\emph{enviroments} - virtualenv -p /usr/bin/python3.6
\textasciitilde{}/venv36 \# Creates a virtual enviroment using a
specific version of python - source
\textasciitilde{}/venv36/bin/activate \# Activates the enviroment - pip
install pandas \# This would install pandas in the enviroment -
deactivate 3. Use Anaconda/Mini-conda - Install Anaconda:
https://anaconda.org/anaconda/python - conda create -n env36 python=3.6
- activate env36 or source activate env36 in linux/mac - install
packages using conda install instead of pip install. Works better in
Windows, as it installs any required DLL
